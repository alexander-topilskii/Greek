# 📖 Правила выноса «урока» из учебника

**[🏠 Readme](../Readme.md) → [🧭 agents.md](../AGENTS.md) → 📄 `docs/lesson-extraction-from-textbook.md`**

*Методичка для авторов репозитория: как связаны сканы, оцифровка и **`essence_N.md`**, как оформлять конспект так, чтобы
читатель **не нуждался во внешних источниках** для повторения темы и мог **отрабатывать готовый материал** без имитации
рабочей тетради.*

## 1. Что такое «урок» в этом репозитории

| Слой             | Файлы                                    | Назначение                                                                                                                                                                                                                                          |
|------------------|------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Скан**         | `lesson_N/raw/*.png`                     | Источник при подготовке материалов: страницы по номеру в колонтитуле.                                                                                                                                                                               |
| **Оцифровка**    | `lesson_N/digitized/K.md` (опционально)  | Текст страницы `K`, формулировки заданий из книги, поиск по уроку.                                                                                                                                                                                  |
| **Навигация**    | `lesson_N/content_N.md`                  | Список страниц, превью, ссылки на `raw` и `digitized`; генерируется скриптом.                                                                                                                                                                       |
| **Суть урока**   | `lesson_N/essence_N.md` (опционально)    | **Конспект для повторения и отработки:** правила, готовые фразы, опорные конструкции, словарь. Это **не** рабочая тетрадь и **не** полный ключ к каждому номеру упражнения — держим **готовые формулировки** и **сжатые эталоны** по правилам ниже. |
| **Voice + HTML** | `lesson_N/essence_N.html` (генерируется) | Не редактировать вручную: собирается из `docs/ai_voice_promt.md` + `essence_N.md` скриптом `scripts/generate_essence_html.py` (вызывается при регенерации `content_N.md`).                                                                          |

**Вынос урока** для авторов — подготовка **`essence_N.md`** и при необходимости оцифровки; **сканы** задают границы
урока по нумерации страниц.

### 1.1. Место `essence` в цепочке «материал → практика»

| Уровень           | Где лежит                                       | Задача                                                                                                                                                                                                                                                        |
|-------------------|-------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Исходник страницы | `raw/K.png`, при необходимости `digitized/K.md` | Точное соответствие печатной странице; формулировка задания из учебника — **в оцифровке**, если она нужна для смысла.                                                                                                                                         |
| Конспект          | `essence_N.md`                                  | **Сценарий повторения:** что учить, в каком порядке отрабатывать (микрофразы → развёрнутые реплики), чеклист «могу ли я». Связь с голосовой отработкой — [ai_voice_promt.md](ai_voice_promt.md) (сначала узлы и короткие конструкции, затем длинные диалоги). |
| Вне репозитория   | Карточки, внешние тренажёры                     | По желанию; в `essence` не дублировать ссылки «пройдите тест на сайте …» как обязательный шаг.                                                                                                                                                                |

Конспект — **опора для повторения вслух и письменно**, а не замена учебнику в виде списка ответов к номерам.

## 2. Границы урока (диапазон страниц)

- Папка **`lesson_N`** = **непрерывный диапазон** страниц, собранный в `raw/`.
- **`K.png`** = номер **печатной** страницы в исходном пособии (процесс переименования и
  проверки: [page-screenshot-renaming.md](page-screenshot-renaming.md); общие правила сканов —
  в [AGENTS.md](../AGENTS.md)).
- В шапке **`essence_N.md`** — **диапазон** (например, «стр. 132–149») и при необходимости **название единицы** в
  кавычках («Ψάχνω για σπίτι»).
- При смене издания или сомнениях **сверяйте цитаты и диапазон** со сканом в `raw/`; номер в колонтитуле страницы —
  ориентир для имени `K.png`.

## 3. Назначение `essence_N.md` (для читателя)

- **Самодостаточность:** всё, что нужно для понимания и повторения темы, **внутри файла** (или в явно указанных файлах
  репозитория — например `digitized/K.md`). **Не** писать «см. учебник», «откройте книгу» как единственный способ понять
  блок.
- Это **не** пересказ каждой страницы подряд и **не** полный сборник решений ко всем упражнениям: фокус на **правилах,
  фразах, лексике** и **эталонах** по правилам раздела 9.
- **Практическая карта темы** и опора для повторения вслух и письменно.
- Не отправляйте читателя к «упр. N на стр. M» как к **единственному** источнику смысла: формулировка задания должна
  быть **в репозитории** (`essence` или `digitized/K.md`), если без неё «голые ответы» бессмысленны.
- **Повторные уроки** (обзор) — карта «что из какого номера урока», единая шпаргалка; **не** переносить в конспект все
  ключи ко всем упражнениям блока — только узловые эталоны и ссылки на оцифровку при необходимости.

### 3.1. Типы уроков (разная строгость)

| Тип                       | Акцент в `essence`                                                                                                                                                        |
|---------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Новая грамматика**      | Таблицы, минимальные пары примеров, опорные фразы с новой конструкцией.                                                                                                   |
| **Обзор**                 | Карта тем по номерам уроков, сжатые эталоны; без «полного ключа» ко всем номерам из раздела повторения.                                                                   |
| **Фонетика / орфография** | Правило + **замкнутые** примеры (пары слов, минимальные контрасты). Не подменять конспект списком «прочитайте все топонимы со страницы», если цель — отработка пары букв. |

## 4. Структура `essence_N.md` (рекомендуемая)

Соблюдайте [AGENTS.md](../AGENTS.md). **Порядок разделов:**

1. Заголовок `#` с эмодзи + тема + диапазон страниц.
2. **Хлебные крошки** до `Readme` → `book/pages` → `content_N.md` → текущий файл.
3. Курсивом — 2–4 предложения **о чём урок** и что файл **самодостаточен** для повторения; **порядок разделов** (
   цели → … → словарь).
4. Таблица **«Быстрые ссылки»**: `content_N.md`, диапазон, соседние `essence_{N±1}.md` или обзорный урок.
5. **`## 📌 …`** — тема и **цели отработки** (см. раздел 8): по возможности **измеримые** формулировки («могу
   построить …», «могу ответить на …»), а не только «повторить тему».
6. **`## 🧠 …`** — грамматика и правила; по желанию краткий подпункт **типичные ошибки** (1–3 для русскоязычных) с
   примером «так нельзя → так можно».
7. **`## 💬 …`** — готовые фразы и диалоги (**полные** реплики для заучивания; без пропусков — см. раздел 6).
8. **`## 📖 …`** *(опционально)* — эталоны для самопроверки — **сжато**, по классификации раздела 9.
9. **`## 📇 …`** — словарь и опоры — **последний раздел** файла; сюда же уместен **итоговый чеклист** «могу ли я…» (
   обязательно держать согласованным с целями из 📌).

**Мета-текст** (как пояснять устройство файла):

- **Один раз** во вступлении: роль файла и порядок разделов.
- В блоке **эталонов** — одна строка: зачем блок (сверка), без повторения длинных отграничений, если мысль уже вверху.
- **Отсылки «см. выше»** — по названию подраздела или якорю; не дублировать большие таблицы.
- Заголовок раздела со словарём уже задаёт место лексики; не дублировать это в курсиве в том же заголовке.

**Не закрепляйте** в тексте конспекта сценарии внешних голосовых ассистентов (сценарий сессии —
в [ai_voice_promt.md](ai_voice_promt.md)).

## 5. Что включить в конспект при выносе из материалов курса

- **Название темы** и **ключевые реплики** из выделенных блоков («Πώς το λένε;», «Για δες»).
- **Таблицы** грамматики в Markdown + краткое правило по-русски.
- **Числительные**, **артикли с предлогами**, **типичные ошибки** — с **примерами** в предложениях.
- **Диалоги** — полный текст в конспекте, если иначе смысл теряется; не оставлять **роль с пропусками** (______) без
  полного эталона рядом или без отсылки на `digitized/K.md`, где есть задание целиком.
- **Один эталон — одна функция:** например пара «вопрос — ответ», жалоба, просьба; не превращать раздел в сплошной
  список «ответ 1, ответ 2» без контекста задания в репозитории.

## 6. Чего избегать в тексте `essence_N.md`

- Отсылок к печатному пособию как к **обязательному** источнику для понимания.
- Смысла, который возможен **только** через «упр. N на стр. M» без расшифровки в конспекте или в **`digitized/K.md`**.
- **Пропусков для вставки** (пробелы, «вставьте форму», фрагменты как в рабочей тетради), если цель файла — отработка *
  *готового** материала: такие фрагменты относятся к учебнику/оцифровке. Исключение: рядом дан **полный заполненный
  вариант** той же сцены (как эталон для чтения и заучивания).
- **Голых ответов** к упражнениям: если в конспекте есть ответ, читатель должен видеть **формулировку задания** в этом
  же файле **или** в явно указанном `digitized/K.md` (ссылка вида «формулировка: [digitized/62.md](digitized/62.md)»).
- **Нумерации «Упр. N — стр. M»** как единственной структуры блока: допустим ориентир по странице, но заголовки по
  смыслу («Спряжение в контексте», «Топонимы») предпочтительнее бесконечного списка номеров из пособия.
- **Домашних заданий в формулировке учебника** («напишите текст на N слов», «выполните в тетради упр. …»): это не
  конспект. Допустима **одна модельная мини-текст** + список опорных конструкций для самостоятельного рассказа — без
  счётчика слов и без имитации ДЗ.
- Полного дублирования **всех** упражнений с пропусками: это размывает фокус конспекта.
- Противоречий исходным материалам: при сомнении — **скан** и **оцифровка** страницы в репозитории.

## 7. Связь с `digitized/K.md`

- Весь материал должен быть самодостаточен и поняетн без `digitized/K.md`.

## 8. Цели и мини-метод отработки

- В **`## 📌`** по возможности формулируйте **проверяемые** цели (что умеет делать ученик после повторения),
  согласованные с финальным чеклистом в 📇.
- **Мини-метод** (рекомендация читателю, можно одной строкой во вступлении): прочитать опорные фразы вслух →
  воспроизвести по памяти короткие блоки → затем развёрнутые реплики и диалоги; длинные диалоги — после микрофраз (
  согласуется с [ai_voice_promt.md](ai_voice_promt.md)).

## 9. Эталоны: термины и лимиты

| Термин               | Смысл                                                                                                                                                         |
|----------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Опорная фраза**    | Короткая готовая конструкция для повторения и вариаций.                                                                                                       |
| **Полный эталон**    | Законченный диалог, абзац или таблица «вопрос — ответ» без пропусков.                                                                                         |
| **Сверка по номеру** | Почти полный ключ к конкретному упражнению — по умолчанию **не** в `essence`; если очень нужно — кратко + ссылка на `digitized/K.md` с формулировкой задания. |

Правило: **не превращать** файл в полный ключ ко **всем** номерам раздела; для обзорных уроков — только узловые примеры
и карта тем.

## 10. Оформление греческого текста

- Внутри одного конспекта держите **единую политику**: нужны ли **ударения** в учебных примерах; **монотоник** vs полный
  **политоник** — выберите один подход и не смешивайте без причины.
- Транслитерация — только там, где помогает читателю; не раздувать строки латиницей.

## 11. Связь с `content_N.md`

- После добавления или переименования `essence_N.md` запустите:

  `python3 scripts/generate_book_lesson_content_md.py`

- В `content_N.md` появятся строки **«💎 Суть урока»** (на `essence_N.md`) и **«🎙 Voice (HTML)»** (на `essence_N.html` и
  общий индекс), если файл конспекта существует. Обновляется также `essence_N.html`
  и [essence_voice_index.html](../book/pages/essence_voice_index.html).

## 8. Контрольный список перед коммитом

- [ ] Диапазон страниц в шапке совпадает с `raw/*.png` в папке урока.
- [ ] Конспект **самодостаточен** для повторения основных блоков темы.
- [ ] Таблицы выровнены, ссылки относительные, стиль согласован с [AGENTS.md](../AGENTS.md).
- [ ] Скрипт генерации выполнен при изменении `essence_*.md` (пересобраны `content_*.md`, `essence_*.html`, индекс Voice).
